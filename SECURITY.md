# Security Policy

## Supported Versions

Only the latest public release is actively maintained.

## Reporting a Vulnerability

Please do not open a public issue for sensitive security problems.

Report privately through the project maintainer or the official website contact channel:

```text
https://www.syapi.vip/
```

## Sensitive Data

Do not submit the following information in GitHub issues, pull requests, screenshots, or logs:

- API Keys
- Bearer tokens
- Server passwords
- SSH private keys
- User `%USERPROFILE%\.codex` files
- Feedback backend data

## Project Safety Boundary

This project does not provide bypass, cracking, antivirus evasion, or security-disabling features.

If an installer is flagged by Windows SmartScreen or antivirus software, the supported remediation path is code signing, HTTPS distribution, vendor false-positive submission, and reputation building.
