# SY Codex v0.2.0 Simple Provider UI Design

## Goal

Build the first v0.2.0 user-facing upgrade around the selected mockup:

`D:/CODEX+/codex-manager/.superpowers/brainstorm/v020-ui-provider/content/final-user-view-v1.html`

The product should feel simple for new users in China: install Codex, choose an API provider, paste a key, fetch upstream models, save the config, and open Codex.

## Success Criteria

- The app presents one simple main operation page, not a complex admin dashboard.
- Users can understand the path without reading technical docs:
  1. Select tool.
  2. Download and install Codex.
  3. Choose API provider.
  4. Paste API Key.
  5. Fetch upstream models.
  6. Save API config.
  7. Open Codex.
- The tutorial panel explains SY API as a 2-discount GPT model aggregation entry.
- The tutorial points users to `www.syapi.com` for recharge, token creation, and key copy.
- API config writes to the correct Codex config directory by system:
  - Windows: `%USERPROFILE%\.codex`
  - Mac: `~/.codex`
- Existing Codex config and auth files are backed up before writing.
- Existing update check, feedback, logging, and Chinese enhancement remain available.

## UI Design

Use `final-user-view-v1.html` as the implementation baseline.

The top area shows:

- Product name: `SY Codex（聚合安装）`
- Short value text: one-click Codex install and proxy API config.
- Current version.
- Chinese enhancement status.

The main area uses a two-column layout:

- Left: user operation cards.
- Right: beginner tutorial.

The left operation cards are:

- `选择工具`
  - User selects Codex.
  - OpenClaw stays visible as a reserved future entry.
  - Do not split Codex download by Windows/Mac in this version.
  - System detection is only used later to show and write the correct API config path.
- `配置 API 供应商`
  - Provider cards: SY API, DeepSeek official, Zhipu official, Custom.
  - Selecting a provider fills base URL, protocol, and default model.
  - API Key remains user input.
  - Buttons: `一键获取上游模型`, `保存 API 配置`.
- `完成配置`
  - Shows final launch action.
  - Button: `打开 Codex 桌面 App`.
  - Shows the simple unsigned-installer explanation:
    `下载后提示“不安全程序”通常是因为安装包没有认证，允许安装即可。`

The tutorial panel includes:

- What SY API is: `2 折 GPT 模型聚合入口`.
- Step 1: open `www.syapi.com`.
- Step 2: recharge.
- Step 3: create API token.
- Step 4: paste Key into the app.
- Step 5: fetch models, choose model, save config.
- Step 6: open Codex.
- Customer service: `weixxxnb`.

## API Providers

Provider presets should be data-driven so adding providers later does not require rewriting the form.

Initial presets:

- SY API
  - Display: `SY API`
  - Base URL: `https://www.syapi.com/v1`
  - Protocol: Responses API
  - Default model: `gpt-5.5`
  - Tutorial: enabled
- DeepSeek official
  - Display: `DeepSeek 官方`
  - Base URL: `https://api.deepseek.com`
  - Protocol: Chat Completions
  - Default model: `deepseek-chat`
- Zhipu official
  - Display: `智谱官方`
  - Base URL: `https://open.bigmodel.cn/api/coding/paas/v4`
  - Protocol: Chat Completions
  - Default model: use the first returned model if available; otherwise keep a conservative preset.
- Custom
  - User edits base URL, protocol, and model.

## Model Fetching

The `一键获取上游模型` button calls the selected provider model-list endpoint.

Expected behavior:

- Validate base URL and API Key before fetching.
- Show loading progress while fetching.
- Parse OpenAI-compatible `/models` responses.
- Populate the model selector.
- If the fetch fails, keep manual model input usable and show a clear message.
- Do not block saving if the provider does not expose model list but the user manually enters a model.

## Config Writing

Reuse the existing backend config writer wherever possible.

Writing behavior:

- Save provider record for app reuse.
- Write `config.toml`.
- Write `auth.json`.
- Backup old files before writing.
- Avoid duplicate `model`, `model_provider`, and `[model_providers.custom]` entries.
- Preserve unrelated user config blocks such as features, desktop settings, projects, marketplaces, and windows sandbox settings.

System path behavior:

- Windows uses `USERPROFILE`.
- Mac uses `HOME`.
- The UI displays the detected path in simple text.
- System detection does not change the Codex download button in this version.

## Error Handling And Logs

Use beginner-friendly messages:

- Missing key: `请先粘贴 API Key。`
- Bad base URL: `Base URL 格式不正确，请检查供应商地址。`
- Model fetch failed: `模型列表获取失败，可以手动填写模型后保存。`
- Config write failed: show the error and tell user to open logs.

Add or keep logs for:

- Detected OS.
- Selected provider.
- Model fetch URL without API Key.
- Model fetch result count or failure reason.
- Config write path.
- Config write success or failure.
- Codex launch attempt.

Never log the full API Key.

## Out Of Scope For This Version

- Mac one-click Codex installation.
- OpenClaw installation.
- Code signing.
- Replacing official Codex installer contents.
- Advanced provider management page.
