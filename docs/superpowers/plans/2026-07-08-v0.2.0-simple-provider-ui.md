# SY Codex v0.2.0 Simple Provider UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the approved beginner-friendly UI with provider presets, upstream model fetching, automatic config-path messaging, tests, build, and GitHub release update.

**Architecture:** Keep the existing Tauri + React wizard, but simplify the visible user flow around Codex install and API configuration. Provider presets live in a small frontend data module; the existing Rust config writer remains the source of truth for `config.toml` and `auth.json`.

**Tech Stack:** React 19, Mantine, TypeScript, Vitest, Tauri 2, Rust.

---

- Modify `src/types.ts`: add provider preset id and optional display metadata.

- Build output: create and publish `SY-Codex_0.2.0_x64-setup.exe`.
